Aula 01 - Introdução ao HTML

Conhecendo o HTMl e sua estrutura básica

  • O que é o HTML e qual é o seu papel na construção de páginas web? Descreva brevemente a estrutura básica de um documento HTML.
  • HTML (HyperText Markup Language) é a linguagem de marcação padrão usada para criar e estruturar páginas web.
    Um documento HTML típico possui uma estrutura básica que inclui os seguintes elementos:
    <!doctype html> Declaração do tipo de documento que informa ao navegador que o documento é um HTML.
    <html>: Elemento raiz que envolve todo o conteúdo da página.
    <head>: Contém metadados sobre o documento, como o título da página, links para arquivos CSS, scripts, e outras informações meta.
    <title>: Define o título da página que aparece na aba do navegador.
    <body>: Contém o conteúdo visível da página, como textos, imagens, links, etc.

  • Como o HTML se relaciona com outras tecnologias da web, como o CSS e o JavaScript? Como essas três tecnologias trabalham em conjunto para criar páginas web interativas e estilizadas?
  • HTML, CSS e JavaScript são as três tecnologias fundamentais para a construção de páginas web. Cada uma delas desempenha um papel específico e, quando usadas em conjunto, permitem a criação de páginas web interativas e estilizadas.
    HTML: Cria a estrutura básica da página.
    CSS: Aplica estilos ao conteúdo HTML.
    JavaScript: Adiciona interatividade à página.

Identificando tags HTML básicas

  • Cite três tags HTML básicas e explique o propósito de cada uma delas. Dê exemplos de situações em que você as usaria ao construir uma página web.
  • <h1> a <h6> (Cabeçalhos)
    Propósito: Definir títulos e subtítulos na página, com <h1> sendo o mais importante e <h6> o menos importante. Exemplo de Uso:
    Usar <h1> para o título principal da página e <h2> a <h6> para subtítulos e seções subsequentes.

  • Além das tags mencionadas, qual é a importância das tags de listagem, como <ul> e <ol>, no contexto da estruturação de conteúdo em uma página? Dê exemplos de quando você escolheria uma sobre a outra.
  • As tags de listagem, como <ul> (lista não ordenada) e <ol> (lista ordenada), são importantes para estruturar e organizar o conteúdo em uma página web de maneira clara e legível. Elas ajudam a agrupar itens relacionados, facilitando a leitura e a navegação para os usuários.
    <ul> (Lista Não Ordenada) Propósito: Criar uma lista de itens onde a ordem não é importante. Exemplo de Uso: Usar <ul> para listas de características, itens de menu, ou qualquer conjunto de itens onde a sequência não importa.
    <ol> (Lista Ordenada) Propósito: Criar uma lista de itens onde a ordem é importante. Exemplo de Uso: Usar <ol> para listas de instruções, passos de um processo, ou qualquer conjunto de itens onde a sequência é relevante.
    Quando Escolher Uma Sobre a Outra?
    <ul> Quando a ordem dos itens não faz diferença. Por exemplo, uma lista de recursos de um software, itens de um menu de navegação, ou uma lista de contatos.
    <ol>: Quando a ordem dos itens é crucial. Por exemplo, uma lista de etapas para completar uma tarefa, uma receita de culinária, ou uma lista de prioridades.

Vinculando páginas, arquivos e âncoras

  • Como a tag <a> é utilizada para criar links em HTML? Explique como você pode vincular para outras páginas, para arquivos locais e criar âncoras dentro de uma página.
  • A tag <a> é utilizada para criar links em HTML. Para vincular a outras páginas, você deve usar o atributo href com o URL da página de destino, como <a href="pagina.html">Link</a>. Para arquivos locais, você pode fornecer o caminho relativo, como <a href="documentos/arquivo.pdf">Arquivo PDF</a>. Para criar âncoras dentro da mesma página, você pode definir um id no elemento alvo e, em seguida, criar um link para ele usando o atributo href com uma hashtag (#), como <a href="#secao">Vá para a seção</a>.

  • Além do atributo href, quais outros atributos importantes a tag <a> pode ter? Como o atributo target pode ser usado para controlar como o link é aberto pelo navegador?
  • Além do atributo href, a tag <a> pode ter outros atributos importantes, como o target e o title. O atributo target é utilizado para controlar onde o link será aberto. Por exemplo, <a href="https://example.com" target="_blank"> abrirá o link em uma nova aba ou janela. O atributo title pode fornecer informações adicionais ao usuário, exibidas quando o cursor passa sobre o link.

Incorporando imagens e mídias

  • Descreva a diferença entre os atributos src e alt em relação à tag <img>. Por que é importante fornecer um atributo alt adequado para imagens?
  • O atributo src especifica a origem da imagem, ou seja, o caminho do arquivo de imagem, como <img src="imagens/foto.jpg">. O atributo alt fornece um texto alternativo que é exibido se a imagem não puder ser carregada e é essencial para a acessibilidade, pois descreve a imagem para leitores de tela usados por pessoas com deficiência visual.

  • Além de imagens, quais outros tipos de mídia podem ser incorporados em uma página HTML? Como você pode garantir que a mídia incorporada seja responsiva e se ajuste bem em diferentes dispositivos?
  • Além de imagens, outros tipos de mídia que podem ser incorporados em uma página HTML incluem vídeos (usando a tag <video>), áudio (usando a tag <audio>), e elementos embutidos como iframes. Para garantir que a mídia seja responsiva, você pode usar CSS para definir largura e altura percentuais, ou utilizar unidades relativas como "em" ou "vw", além de técnicas como o uso de media queries para ajustar o layout em diferentes tamanhos de tela.

Criando tabelas básicas

  • Quais são as principais tags usadas para criar tabelas em HTML? Explique a diferença entre as tags <th> e <td>, e quando você as usaria.
  • As principais tags usadas para criar tabelas em HTML são <table>, <tr>, <th>, e <td>. A tag <table> define a tabela, <tr> define uma linha na tabela, <th> define uma célula de cabeçalho (que geralmente aparece em negrito e centralizada), e <td> define uma célula de dados. Usaria <th> para cabeçalhos de colunas ou linhas, e <td> para todas as outras células de dados.

  • Embora tabelas sejam usadas para exibir dados tabulares, às vezes é recomendado evitá-las em favor de outras formas de apresentação de dados. Por quê? E quais são os cenários em que as tabelas são apropriadas?
  • Tabelas são ótimas para exibir dados tabulares organizados em linhas e colunas. No entanto, é recomendado evitá-las em casos onde a estrutura dos dados não seja essencialmente tabular ou quando a tabela poderia ser substituída por listas, gráficos, ou layouts de grid que proporcionem uma experiência visual mais atraente e acessível.
    Isso ocorre porque tabelas podem ser difíceis de adaptar para layouts responsivos e podem apresentar problemas de acessibilidade, especialmente para usuários de leitores de tela.
    As tabelas são apropriadas quando os dados precisam ser organizados em uma grade, onde a relação entre os dados nas linhas e colunas é importante. Exemplos incluem relatórios financeiros, horários de aula, ou qualquer conjunto de dados onde a comparação direta entre diferentes categorias é necessária.

Criando formulários

  • Como a tag <form> é usada em HTML para criar formulários? Explique a função dos atributos action e method no contexto de um formulário.
  • A tag <form> é usada em HTML para criar formulários, que permitem a entrada e envio de dados do usuário. O atributo action especifica o URL para onde os dados do formulário devem ser enviados ao serem submetidos, e o atributo method define o método HTTP a ser usado, geralmente "GET" ou "POST". "GET" anexa os dados à URL, enquanto "POST" envia os dados de forma mais segura e não visível na URL.

  • Além dos campos de entrada, quais outros elementos podem ser incluídos em um formulário para melhorar a usabilidade e a experiência do usuário?
  • Além dos campos de entrada <input>, vários outros elementos podem ser incluídos em um formulário para melhorar a usabilidade e a experiência do usuário:

    1. <label>: Associado a um campo de entrada, o elemento <label> melhora a acessibilidade e ajuda o usuário a entender o propósito do campo.

    2. <select> e <option>: Permitem a criação de menus suspensos, permitindo que os usuários escolham uma opção a partir de uma lista predefinida.

    3. <textarea>: Usado para permitir que os usuários insiram grandes blocos de texto, como em comentários ou descrições.

    4. <button>: Usado para acionar ações como enviar o formulário, redefinir campos, ou iniciar outras interações.

    5. <fieldset> e <legend>: Agrupam campos relacionados e fornecem uma legenda para o grupo, ajudando a organizar o formulário de maneira clara.

Técnicas avançadas de estruturação com elementos HTML5 semânticos

  • O que são elementos HTML5 semânticos e por que são importantes na estruturação de uma página? Cite três exemplos de elementos semânticos e explique quando você os usaria.
  • Elementos HTML5 semânticos são aqueles que claramente descrevem o significado de seu conteúdo tanto para navegadores quanto para desenvolvedores. Eles são importantes porque melhoram a acessibilidade, a indexação por motores de busca, e a clareza do código, tornando a estrutura da página mais intuitiva e fácil de entender.

    Três exemplos de elementos semânticos:

    1. <header>: Usado para definir a seção de cabeçalho de uma página ou de uma seção de conteúdo. É ideal para incluir logotipos, menus de navegação, e outras informações introdutórias.

    2. <article>: Utilizado para conteúdo independente, como posts de blogs, artigos de notícias, ou qualquer conteúdo que possa ser distribuído de forma independente ou reutilizável.

    3. <footer>: Define o rodapé de uma página ou seção. Normalmente contém informações sobre o autor, links de navegação, e direitos autorais.

  • Além da semântica, como os elementos HTML5 podem afetar a acessibilidade das páginas web? Por que é fundamental considerar a acessibilidade ao escolher elementos de estruturação?
  • Além da semântica, os elementos HTML5 desempenham um papel crucial na acessibilidade das páginas web, pois ajudam tecnologias assistivas, como leitores de tela, a interpretar e navegar corretamente pelo conteúdo. Elementos semânticos informam aos dispositivos de acessibilidade a função de diferentes partes da página, permitindo uma experiência de navegação mais fluida para pessoas com deficiências.

    Considerar a acessibilidade ao escolher elementos de estruturação é fundamental porque garante que todos os usuários, independentemente de suas habilidades físicas ou cognitivas, possam acessar e utilizar o conteúdo da página. A inclusão de elementos semânticos adequados facilita a navegação, melhora a compreensão do conteúdo, e cumpre requisitos de conformidade com normas de acessibilidade, como a WCAG (Web Content Accessibility Guidelines).
